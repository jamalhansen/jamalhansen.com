## Why This Caught My Eye

The argument is that code you vibe-coded becomes legacy code the moment you move on, because nobody understands it well enough to maintain it confidently. That includes you, two months later.

This sits in direct tension with what I write about in the "Vibe Coded and Lived to Tell" series. My take has been that vibe coding is a valid learning ramp and a legitimate prototyping tool. I think both things are true: it gets you moving fast, and the output is fragile in a specific way that earns the legacy label. Interesting read.
